# Step 7 — CI/CD & Deployment Config (DEPLOY) [OPTIONAL]

> **Trigger:** `/d2c deploy`
> 
> **Note:** This step is fully optional. The core D2C workflow ends at Step 5 (or Step 6 if user opted into code generation via `/d2c code`). Deployment configuration is an add-on triggered by `/d2c deploy`. See `.d2c/PLAYBOOK.md` for deployment prerequisites.

**Input:** Completed code + AGENTS.md (deployment plan)
**Output:** Deployable app + `README.md`

## 7a. Deployment Config

**If the directory is empty (no existing project):** Write config files per Step 3 decisions.

**If an existing project was detected:** Check if config exists (`vercel.json` / `Dockerfile` /
`.github/workflows/`). Reuse if present; create only if missing.

**Vercel:**
```bash
echo '{"framework":"nextjs","buildCommand":"npm run build","outputDirectory":".next"}' > vercel.json
```

**Railway:** Create `railway.json` + `nixpacks.toml`.

**Docker:** Create `Dockerfile` + `docker-compose.yml`.

**Cloudflare Pages:** Create `wrangler.toml`.

## 7b. CI/CD Pipeline

**GitHub Actions:**
```yaml
# .github/workflows/deploy.yml
name: Deploy
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npm run build
      # deployment steps per platform
```

## 7c. Environment Variables

Create `.env.example` with all required variables and descriptions.

## 7d. Build Verification

Run `npm run build` (or equivalent). Fix any build errors.

## 7f. Finalize Root AGENTS.md ⭐

The root `AGENTS.md` (generated in Step 3) is the project context file
auto-detected by Claude Code, Cursor, Windsurf, and other AI coding tools.
Append or update the following section(s) that were undetermined at Step 3:

### 7f-1. Append Deployment Section

Add deployment details if they weren't included in Step 3 (e.g., because the
deploy target was pending user confirmation):

```markdown
## Deployment
- **Target:** {platform}
- **Build:** `npm run build`
- **Config:** {config file if any}
- **Domain:** {if known}
- **Env vars required:** {list known vars, template in .env.example}
```

### 7f-2. Verify Constraint References

Ensure the `## Constraints` section of `AGENTS.md` correctly references all
generated `.d2c/` documents:

```markdown
## Constraints
- Design system + tokens + behavioral rules → `.d2c/DESIGN.md`
- Component tree + coding + Token Adoption rules → `.d2c/SPEC.md`
- Design token source → `.d2c/DESIGN.md`
```

**Do not duplicate detailed constraint content** into AGENTS.md — the root
file is the index, `.d2c/` is the library. Keeping it lean (~40 lines max)
ensures tools load it as context without wasting tokens.
